# 屏幕功能规格

每个屏幕包含:**进入条件 / 核心信息 / 用户操作 / 异常状态**

---

## Screen 01:Splash 启动屏

- **进入条件**:每次打开小程序的第一个画面
- **停留时间**:< 1.5 秒,数据预加载完成立刻跳转
- **核心信息**:
  - 渐变蓝色海洋背景
  - Logo "🎣 鲷会"
  - 副标 "TideTai · for the angler"
  - 加载中指示器
- **下一步**:
  - 首次打开(本地无 user_settings)→ Onboarding(屏 2)
  - 已有 user_settings → 主屏(屏 4)

---

## Screen 02:Onboarding 引导介绍

- **进入条件**:首次启动,或用户在设置里手动重置
- **核心信息**:
  - 主标题:"出海前的 30 秒,看一眼这里"
  - 副标:"鲷会把潮汐、风、天气、气压、月相整齐地摆在你面前。判断由你做,信息由我们给。"
  - 4 个 Bullet Point:
    - 🌊 本地化潮汐 · 精确到分钟的涨落
    - 🌬️ 风对你的钓点是顺还是背
    - 📝 渔获自动归档 · 一键朋友圈
    - 📵 照片不上云 · 只在你的手机里
- **操作**:
  - 「下一步」→ 钓点选择(屏 3)
  - 「跳过」→ 直接进入主屏(默认无钓点状态)

---

## Screen 03:Onboarding 钓点选择

- **进入条件**:从引导介绍点击「下一步」
- **核心信息**:
  - 调用 `wx.getLocation` 获取用户位置
  - 后端返回附近 5-10 个钓点(按距离排序)
  - 每个钓点显示:名称、距离、钓法
  - 多选(≥ 1 个)
- **操作**:
  - 选中/取消选中钓点
  - 「+ 找不到?手动添加 / 在地图上选」→ 跳到地图选点页
  - 「完成,开始使用」→ 主屏
- **异常状态**:
  - 用户拒绝定位授权 → 显示一段说明,引导用户手动搜索钓点
  - 用户没有可选钓点(冷启动地区数据不足)→ 显示「找不到附近钓点?手动添加」CTA

---

## Screen 04:主屏(每日决策面板)⭐

- **进入条件**:从 Splash 或返回主屏
- **核心信息**:
  - 顶部:钓点(可切换 ▾) + 日期 + 大小潮标签 + 数据更新时间戳
  - 潮汐:24 小时连续波形(SVG) + 当前时间标记(红色虚线) + 高低潮关键点(时间 + 潮高)
  - 风:风向、风力、阵风、对当前钓点是「顺/背/侧」风
  - 天气:状态描述、气温区间、水温
  - 气压:数值 + 24 小时趋势(↑ 上升 X hPa / ↓ 下降 X hPa)
  - 月相:文字描述(满月前/后 N 天) + 月出时间
  - 「点击查看完整数据」入口(二级页,展示日出日落、常见鱼种等次要数据)
  - 底部 4 个导航:7 日 / 记一笔 / 钓点 / 我的
- **操作**:
  - 顶部钓点 ▾ → 弹出钓点切换菜单
  - 下拉刷新 → 重新拉数据
  - 点击「点击查看完整数据」→ 详情页
- **异常状态**:
  - 数据加载中 → 骨架屏(屏 10)
  - 网络失败 → 黄色横幅警告 + 显示缓存数据 + 重试按钮(屏 10)

### 设计原则

- **严格一屏可见**(以 iPhone 14 Pro 393pt 宽度为准)
- 主屏不要超过一屏高度,任何不重要的信息都进入二级页

---

## Screen 05:七日预报

- **进入条件**:主屏底部点「7 日」
- **核心信息**:每行一天,从左到右:
  - 日期(今天/明天/周几 + 月日)
  - 大小潮标签(色块:深蓝大潮 / 中蓝中潮 / 浅蓝小潮)
  - 当日潮汐迷你波形图(SVG)
  - 风向风力(异常风力红色标记)
  - 天气 emoji
  - 最高气温
- **操作**:
  - 点击任一天 → 跳到该日期的详情(主屏视图但日期切换为该日)
  - 「‹ 返回」→ 主屏
- **异常状态**:同主屏

---

## Screen 06:钓点管理

- **进入条件**:主屏底部点「钓点」
- **核心信息**:
  - 顶部统计:已收藏 N 个
  - 钓点列表:每项显示
    - 钓点名称
    - 距离 + 钓法
    - 最近一次出海记录(或「还没去过」)
  - 当前激活的钓点高亮标识「当前」
  - 「+ 添加新钓点」CTA
- **操作**:
  - 点击钓点 → 设为当前钓点 + 返回主屏
  - 点击右上「编辑」→ 进入排序/删除模式
  - 「+ 添加新钓点」→ 地图选点页 / 搜索页

---

## Screen 07:渔获记录表单

- **进入条件**:主屏底部点「记一笔」
- **核心信息**:
  - 钓点/时间(从主屏继承,可编辑)
  - 渔获鱼种(chip 标签 + 数量,可添加多条)
  - 渔获照片(可选,最多 9 张)
    - 点击「+ 拍照」→ 调用 `wx.chooseMedia`
    - 照片只在内存中,**不传服务器**
    - 提示文案:「📵 照片只保存在你的手机本地,不会上传服务器」
  - 尺寸/重量(可选)
  - 用饵(预设 chip + 「其他」手动输入)
  - 备注(可选,textarea)
  - 「系统已自动记录」横幅:潮汐/水温/风向/月相/气压/气温
  - 「保存并生成渔获卡」按钮
- **操作**:
  - 「保存并生成渔获卡」→
    1. 把表单数据 + 自动捕获的天气/潮汐快照,POST 到后端保存
    2. 跳转到渔获卡预览页(屏 8),传递照片(本地路径)和数据
- **异常状态**:
  - 用户拒绝相机授权 → 显示说明 + 引导去设置开启
  - 网络失败保存数据时 → 暂存本地,提示「将在网络恢复后同步」

---

## Screen 08:渔获卡预览 + 分享

- **进入条件**:从渔获记录表单保存后
- **核心信息**:Canvas 合成的渔获卡图片
  - 顶部封面:用户照片(渐变背景 + 错位拼接 ±2°)
  - 中部:迷你潮汐波形 + 作钓时段高亮(金色色块)
  - 鱼种列表(数量 + 最大尺寸)
  - 数据小方框:用饵、水温、气压、月相
  - 底部:第 N 次出海 · 累计 X 种鱼 + "鲷会" 品牌
- **操作**:
  - 「‹ 返回修改」→ 回到表单
  - 「⚙️ 隐私」→ 弹出每个字段的显示开关(用饵、重量等可隐藏)
  - 「保存到相册」→ `wx.canvasToTempFilePath` + `wx.saveImageToPhotosAlbum`
  - 「发给朋友」→ `wx.showShareMenu` (分享给好友)
  - 「朋友圈」→ 引导用户从相册分享(微信小程序限制,不能直接分享朋友圈)
- **关键技术点**:
  - Canvas 合成必须在小程序内完成,绝不上传后端
  - 合成尺寸建议 750×1000 px,适合朋友圈竖图

---

## Screen 09:我的渔获列表

- **进入条件**:主屏底部点「我的」
- **核心信息**:
  - 顶部统计条:出海次数 / 鱼种数 / 总尾数
  - 时间线列表,每项:
    - 缩略图(三种状态)
      - 有照片 → 显示照片
      - 无照片(可能换手机丢了)→ 「📷 无照片」占位
      - 空军记录(0 条)→ 海浪图标,文字灰色
    - 钓点名 + 相对时间(今天/N 天前)
    - 鱼种 + 数量(空军则显示「空军 · 0 条」)
    - 当时的潮汐/风/用饵
- **操作**:
  - 点击任一记录 → 进入该记录的详情页(展示完整数据 + 渔获卡)
- **数据加载**:分页,每页 20 条

---

## Screen 10:加载状态 + 错误状态

### 加载(骨架屏)

- 主屏数据加载中显示
- 形状仿真:顶部、潮汐图、4 个数据块都用灰色矩形占位
- 配 1.5s 的脉动动画

### 错误(网络异常)

- 黄色横幅:「⚠️ 网络连接失败」+「显示的是 X 小时前的缓存数据」+「点击重试」按钮
- 缓存数据透明度降低(opacity 0.6),提示用户数据非实时
- 数据更新时间戳变红色「⚠️ 6 小时前缓存」
